# APPLE HUMAN INTERFACE GUIDELINES - DEEP RESEARCH EXTRACTION

## 1. CORE DESIGN PRINCIPLES

### The Three Pillars of Apple Design

**Clarity**
- Legible text at every size
- Precise and lucid icons
- Focused approach eliminating ambiguity
- Text should be readable, icons should be clear, and functionality should be obvious
- Emphasizes creating a seamless user interface

**Deference**
- Interface facilitates content discovery without distraction
- UI helps users understand content but never competes with it
- Fluid animations and translucent/subtle UI elements
- Content takes center stage
- UI steps back to let user content shine

**Depth**
- Visual layering and spatial depth
- Smooth transitions create sense of location
- Visually significant transitions convey hierarchy and meaning
- Users understand where they are and how they got there
- Logical hierarchy creates intuitive navigation

**Additional Universal Principle:**
- Consistency: Ensuring familiarity and predictability across the platform

## 2. ORGANIZATIONAL STRUCTURE

Apple HIG is divided into six main categories:

1. **Platforms** - iOS, iPadOS, macOS, tvOS, visionOS, watchOS
2. **Foundations** - Color, layout, typography, icons, materials, motion, accessibility
3. **Patterns** - Sharing, multitasking, search, feedback, common user experiences
4. **Components** - System components with consistency guidelines
5. **Inputs** - Touch, keyboards, gestures, eye gaze (visionOS)
6. **Technologies** - AirPlay, Apple Pay, ARKit, HealthKit, Siri integration

## 3. TYPOGRAPHY SYSTEM

### Font Family: SF Pro (San Francisco)

**Variants:**
- SF Pro Display: Use at 20pt or higher
- SF Pro Text: Use for body text and smaller sizes
- SF Pro Rounded: Alternative rounded variant
- SF Mono: Monospaced variant
- New York: Serif system font
- 9 weights available (including italics)
- Features: small caps, fractions, inferior/superior numerals, indices, arrows

### Text Style Hierarchy (Semantic Scale)

| Text Style                   | Size     | Usage                                      |
| ---------------------------- | -------- | ------------------------------------------ |
| Large Title                  | 34pt     | Biggest text on iPhone, top-level headings |
| Title 1                      | (varies) | Section headings                           |
| Title 2                      | (varies) | Subsection headings                        |
| Title 3                      | (varies) | Tertiary headings                          |
| Headline                     | (varies) | Emphasized content                         |
| Body                         | 17pt     | Default/primary content                    |
| Callout                      | (varies) | Secondary descriptions                     |
| Subhead                      | (varies) | Supporting text                            |
| Footnote                     | 13pt     | Labels and secondary content               |
| Caption 1                    | (varies) | Image captions                             |
| Caption 2                    | (varies) | Tertiary labels                            |
| Navigation Title (collapsed) | 17pt     | Nav bar text                               |
| Tab Bar Labels               | 10pt     | Smallest iOS interface text                |

**Typography Best Practices:**
- Minimum font size: 11pt for optimal legibility
- Default body text: 17pt
- Maintain relative hierarchy through font weight, size, and color
- Use predefined text styles for consistency and accessibility
- Supports Dynamic Type for accessibility

### Dynamic Type System

**12 Content Size Categories:**

Standard sizes (7 categories):
- xSmall
- Small
- Medium
- Large (Default)
- xLarge
- xxLarge
- xxxLarge

Accessibility sizes (5 additional):
- AX1
- AX2
- AX3
- AX4
- AX5

Location: Settings → Accessibility → Display & Text Size → Larger Text

| secondarySystemFill  | < 1   | Medium-size shapes               |

## 5. SPACING AND LAYOUT SYSTEM

### 8pt Grid System

Apple uses multiples of 8 for spacing:
- 8pt: Minimum spacing increment
- 16pt: Standard spacing between elements
- 24pt, 32pt, 40pt, 48pt, 56pt: Larger spacing increments

**Best Practice:**
- Minimum padding between elements: 8pt
- Recommended standard padding: 16pt in compact, 20pt in regular size class

### Touch Targets

| Platform   | Minimum Size |
| ---------- | ------------ |
| iOS/iPadOS | 44×44 pt     |
| visionOS   | 60×60 pt     |

**Critical Rule:** All tappable elements must meet minimum touch target size for accessibility.

### Safe Area Insets

**iPhone SE:** Status bar only
**iPhone X+:** Top, bottom (portrait); Left, right, bottom (landscape)

**Landscape mode (notched devices):**
- Top: 44pt
- Left/Right: 64pt
- Bottom: 70pt

**Layout Margins:**
- Compact size class: 16pt
- Regular size class: 20pt
- Auto-account for safe area by default (iOS 11+)

### Corner Radius

| Element          | Specification                                                   |
| ---------------- | --------------------------------------------------------------- |
| App Icons        | Superellipse (squircle): radius = side × 0.222, smoothing = 61% |
| Large Cards      | ~20pt                                                           |
| Small Graphics   | ~10pt                                                           |
| 60px icon        | 13px radius                                                     |
| Standard Buttons | Match other UI buttons (default 44pt height)                    |

**Formula for icon radius:** 10/57 × icon size
- Example: 114px icon = 20px radius

## 7. INTERACTION STATES

### Standard States (All Interactive Elements)

| State          | Description                              |
| -------------- | ---------------------------------------- |
| Default        | Normal, untouched state                  |
| Hover          | Mouse cursor over element (iPadOS/macOS) |
| Active/Pressed | Currently being interacted with          |
| Focused        | Selected via keyboard/remote             |
| Disabled       | Not currently interactive                |
| Selected       | Chosen/toggled on                        |

**Apple Approach:**
- Rounded corners, flat design
- Simple color changes or subtle shading
- Less flashy than other platforms
- ARIA-disabled: true for disabled states

**iOS Hover Caveat:**
- iOS applies hover state on tap, persists until another element gets focus
- Not true hover like desktop (no mouse cursor)

## 8. MATERIALS & DEPTH

### Materials (Visual Effects)

Materials create sense of depth, layering, and hierarchy:
- Glass transparency
- Blur levels
- Translucency
- Elevation tokens

**2025 Update: Liquid Glass**
- New material language in iOS 26
- Emphasizes translucency, depth, fluid responsiveness
- Most significant visual redesign since 2013

### Shadow & Blur

**Shadow Implementation:**
- Blur things at lower UI levels (different from Material Design elevation)
- shadowRadius affects edge blur
- Formula: shadowRadius = blur/2 (to match Figma)

**Note:** Specific numerical elevation/shadow specifications not publicly documented in detail.

## 9. NAVIGATION PATTERNS

### Flat Navigation (Tab Bar)

**Characteristics:**
- Divides hierarchy at root level
- 2-5 tabs total
- Always visible (except during modals)
- Tabs should switch views, not trigger actions or sheets
- Predictable behavior required

### Hierarchical Navigation (Navigation Bar)

**Large Title Behavior:**
- Helps orientation during navigation/scrolling
- Transitions to standard title on scroll down
- Returns to large title on scroll to top
- Used by Phone (clarify active tab), Music (differentiate areas)

### Modal Navigation

**When to Use:**
- Single screen or linear series of screens
- Minimal interaction required
- User must complete or explicitly dismiss
- Blocks parent interface
- Requires explicit dismissal action

**Sheet Presentations:**
- Bottom sheets with detents (medium ~50%, large ~100%)
- Perfect for tasks within context
- Minimal/no text input ideal
- Close symbols used sparingly

## 10. ACCESSIBILITY FOUNDATIONS

### Dynamic Type
- 12 content size categories (7 standard + 5 accessibility)
- Out-of-box support with system text styles
- UIFontMetrics for custom fonts
- Scales automatically with user preferences

### Touch Targets
- Minimum 44×44 pt (iOS/iPadOS)
- Minimum 60×60 pt (visionOS)
- Supports finger, pointer, eyes, remote

### Semantic Colors
- Auto-adapt to Dark Mode
- Support high-contrast variants
- Always legible UI
- 4 variants per color minimum

### Screen Reader Support
- Semantic text styles for better compatibility
- ARIA attributes for web content
- VoiceOver optimization

## 11. UNIVERSAL PATTERNS WORTH ADOPTING

### Content-First Philosophy
- UI defers to content
- Eliminate decorative distractions
- Clear visual hierarchy
- Purposeful motion/animation

### Semantic Naming
- Colors by purpose (not hue): label, systemBackground, fill
- Text styles by hierarchy: body, headline, footnote
- Sizes by content: xSmall, medium, large, accessibility sizes

### Progressive Disclosure
- Show relevant information when needed
- Hide complexity until necessary
- Clear navigation breadcrumbs
- Spatial depth conveys location

### Consistency Over Novelty
- Use system components when possible
- Predictable behavior patterns
- Familiar interactions
- Platform conventions respected

### Adaptive Design
- Light/dark mode by default
- High-contrast support
- Dynamic Type support
- Size class adaptivity (compact/regular)
